3. DETAIL HASIL PENGUJIAN
--------------------------------------------------------------------------------
A. PENGUJIAN SINTAKS (CODE PARSING & SYNTAX VALIDATION)
   Metode: Pemindaian iteratif pada seluruh file PHP menggunakan PHP CLI (-l flag)
   Hasil Temuan:
   - File Modul 'Admin': LULUS (Semua file admin/*.php tidak terdapat Parse Error)
   - File Modul 'Petugas': LULUS (Semua file petugas/*.php terbebas dari Missing Brackets / Undefined functions)
   - File Modul 'Anggota': LULUS (Script anggota berhasil dirender tanpa hambatan compile-time)
   - Helper & Config: LULUS (File upload_helper.php & database.php mereturn nilai yang benar sesuai ekspektasi)

B. PENGUJIAN KEAMANAN SISTEM & AKSES (SECURITY TESTING)
   1. SQL Injection Prevention:
      - Hasil: LULUS SANGAT BAIK. Seluruh query database sudah dienkapsulasi menggunakan "Prepared Statements" (bind_param) sehingga manipulasi URL via GET / POST (cth: '?id=1 OR 1=1') akan digagalkan oleh MySQLi.
   2. Cross-Site Scripting (XSS):
      - Hasil: LULUS. Output string di tampilan telah di-filter secara konsisten menggunakan 'htmlspecialchars()'.
   3. Kriptografi & Kata Sandi:
      - Hasil: LULUS. Sistem menggunakan 'password_hash()' (Bcrypt) untuk merubah password pengguna menjadi unreadable ciphertext, memitigasi serangan pencurian akun jika terjadi Dump Database.
   4. Enkapsulasi Level Akses:
      - Hasil: LULUS. Admin tidak dapat mengakses halaman Anggota, begitu pun sebaliknya. Sesi (Session) dijaga ketat oleh file 'includes/session.php', setiap tindakan diblok bila bukan merupakan otorisasi role-nya.

C. PENGUJIAN MODUL DAN FITUR INTI (FUNCTIONAL TESTING)
   1. Modul Manajemen Buku & Kategori:
      - Input Data Baru: Berhasil (stok tervalidasi).
      - Image Upload Protection: LULUS. Pengunggahan sampul buku ('cover') di 'buku.php' tidak hanya mengecek nama file (.jpg/.png), melainkan juga membaca MIME Type melalui *finfo*, memastikan file beresiko (seperti file .php yang disembunyikan/RCE) tidak akan diterima oleh server.
   2. Modul Manajemen Transaksi (Peminjaman & Pengembalian):
      - Kalkulasi Terlambat & Denda: Berhasil. Algoritma mengenali dan menghitung batas jatuh tempo, memperhitungkan Rp. 1.000 per hari keterlambatan.
      - Logika Stok Inventori: Berhasil. Pengurangan stok (-1) ketika peminjaman dan pengembalian aktualisasi stok (+1) tereksekusi bersamaan.
   3. Manajemen Pengguna:
      - Skenario "Tambah Admin Baru oleh Admin": LULUS uji pencegahan. Logic bisnis mencegah duplikasi akun root bersarang (terkonfirmasi diblokir dengan pemaksaan value 'petugas').

D. PENGUJIAN ANTARMUKA & EKSTERNAL (UI/UX & REPORTING)
   1. Kompatibilitas Perangkat & Rendering
      - Browser (Chrome/Edge): Tampilan modern glassmorphism flex layout terjaga sangat baik.
      - Navigasi Datatables: Tombol Sorting, Pagination, dan fungsi 'Search' berjalan responsif di sisi klien.
   2. Mesin Pencetak (A4 PDF Engine Print)
      - Hasil: LULUS SEMPURNA. 
      - Detail Event: Saat tombol klik "Print", engine javascript (EventListener beforeprint) akan mengaktifkan mode 'Show-All Entries'. CSS menghilangkan sidebar, mengatur formasi Grid, dan melebarkan orientasi halaman ke 'Landscape' secara spesifik pada modul-modul denda dan buku yang tabelnya lebar. Segera setelah jendela print tertutup, state tabel akan kembali rapi ke halaman semula (len=10) via 'afterprint'. 

E. PENGUJIAN OTOMATISASI DAN CRON (BACKGROUND JOBS)
   - Background Reminder (Pengingat H-1): Logic sudah tersedia dan terstandarisasi untuk mengirim pesan email melalui protokol SMTP secara asinkron tanpa menunda loading halaman user.
   - Status: SIAP EKSEKUSI jika server/CPanel di-online-kan.
